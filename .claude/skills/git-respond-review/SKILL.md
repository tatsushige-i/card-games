# PR レビューコメント対応スキル

PR のレビューコメントを取得→トリアージ→コード修正→品質チェック→プッシュ→コメント返信を一括実行する。

## 前提条件

- 現在 feature ブランチにいること（main ブランチでは実行しない）
- 対象 PR にレビューコメントが存在すること

## 実行プロセス

### Step 1: PR 特定

1. `$ARGUMENTS` が指定されている場合はその番号を PR 番号として使用する
2. 未指定の場合は `gh pr view --json number` で現在のブランチの PR を自動検出する
3. PR が見つからない場合は **エラーとして中止** する

### Step 2: コメント取得

1. `gh api repos/{owner}/{repo}/pulls/{pr_number}/comments` で全レビューコメントを取得する
2. レビューコメントが存在しない場合は **エラーとして中止** し、「レビューコメントがありません」と案内する

### Step 3: 指摘分析・トリアージ

各コメントについて以下を実施する:

1. コメントの対象ファイル・行番号を特定する
2. 対象ファイルの該当箇所を読み、指摘内容を理解する
3. 指摘の妥当性を判断する:
   - **対応する**: コードに実際の問題があり修正が必要なもの（バグ、型安全性、アクセシビリティ、パフォーマンス等）
   - **対応しない**: 誤検知、好みの問題、既に対応済み、プロジェクト方針と合わないもの等
4. トリアージ結果（対応/非対応とその理由）をユーザーに **一覧表示** する
5. ユーザーの確認を待つ（必要に応じてトリアージ結果を調整）

### Step 4: コード修正

1. トリアージで「対応する」と判断した指摘のみ修正を実施する
2. 対応する指摘がない場合は Step 6 に進む（コメント返信のみ）

### Step 5: 品質チェック

1. quality-checker エージェントで lint・テスト・ビルドを実行する
2. 失敗した場合は修正を行い、再度品質チェックを実行する

### Step 6: コミット＆プッシュ

1. 修正がある場合のみコミット＆プッシュを実行する
2. コミットメッセージ: `fix: Copilotレビュー指摘N件を対応`（N は対応した指摘の数）

### Step 7: コメント返信

全コメントに `gh api` で返信する:

- **対応した指摘**: 修正内容を簡潔に記載
- **対応しない指摘**: 対応しない理由を記載
- 返信の末尾に `🤖 Responded by Claude Code` を付与する
